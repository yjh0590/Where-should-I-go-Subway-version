Ver 1 · 셀룰러/Wi-Fi 측위
GPS 위성 대신 기지국·Wi-Fi 신호로 위치를 잡아요. 지하에서도 대략적인 위치가 나오지만 오차가 수백 m까지 커질 수 있어서, 역 판정 반경을 넓게(600m) 잡고 "연속 확인" 횟수를 늘려 오탐을 막는 전략이에요. 브라우저에서는 저정밀 모드 요청으로 네트워크 측위를 유도합니다.
Ver 2 · 열차 실시간 운행 API
내 폰의 위치가 아니라, "지금 이 노선의 열차들이 어느 역에 있는지"를 알려주는 공개 API를 폴링해요. 지하 여부와 무관하게 정확하다는 게 최대 장점. 여기서는 실제 API 대신 모의 API 서버를 붙여서 응답 형식과 흐름을 그대로 재현했어요. (실제 연동 시: 서울 열린데이터광장 실시간 위치정보, 인천교통공사 API — 키 발급 필요)
폴링 주기: 2초 · 내가 탄 열차를 자동 매칭한다고 가정
API 응답 로그가 여기에 표시돼요
Ver 3 · 타이머 추정 (Dead Reckoning)
GPS가 잘 잡히는 동안은 그대로 쓰다가, 신호가 6초 이상 끊기면 "마지막으로 확인된 위치 + 역간 평균 소요시간"으로 진행 상황을 추정해요. 도착 예상 시간이 지나면 도착으로 간주하고, 신호가 돌아오면 실측으로 즉시 보정합니다. 시뮬레이션에서 📡 버튼으로 터널 진입(신호 끊김)을 재현해보세요.
